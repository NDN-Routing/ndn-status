Brief Overview of NDN

The Named Data Networking (NDN) architecture represents a fundamental paradigm shift from the current Internet architecture. IP identifies data using its location (an IP address). In NDN, the data consumer sends out an Interest packet, which identifies the data that the consumer is seeking by name. The response to an Interest packet in NDN is called a Data packet, which carries both the full name and the data itself.

OSPFN

In order to provide name-based routing capabilities in NDN, Open Shortest Path First (OSPF) has been extended to distribute name prefixes and calculate routes to name prefixes. This protocol is called OSPFN, and is currently deployed on the NDN testbed. The status page allows us to see whether name prefixes are being displayed.
